# 評価実験計画書

## 目的

本ドキュメントでは、開発したQRコード対応多言語注文システムの有効性を検証するための評価実験計画を定義します。留学生による試用評価とアンケートを通じて、システムの実用性と翻訳精度を定量・定性的に評価します。

---

## 1. 評価実験の全体像

### 1.1 実験目的

| 目的 | 説明 | 測定方法 |
|------|------|---------|
| **翻訳精度の検証** | 専門用語辞書による翻訳精度向上効果 | 定量評価（精度比較） |
| **ユーザビリティ検証** | 外国人にとっての使いやすさ | 定性評価（アンケート） |
| **システム有用性検証** | 実際の飲食店で使える可能性 | 定性評価（インタビュー） |

### 1.2 評価の種類

| 評価種別 | 内容 | データ収集方法 |
|---------|------|--------------|
| **定量評価** | 翻訳精度、タスク完了時間、満足度スコア | 実験データ、アンケート（5段階評価） |
| **定性評価** | 使用感、改善要望、問題点 | 自由記述アンケート、インタビュー |

## 2. 評価実験の設計

### 2.1 実験参加者

#### 募集対象

| 属性 | 要件 | 想定人数 |
|------|------|---------|
| **留学生（英語圏）** | 母語が英語、日本語は初級〜中級 | 2-4名 |
| **留学生（中国語圏）** | 母語が中国語、日本語は初級〜中級 | 3-4名 |
| **合計** | - | **5-8名** |

**参加者数削減の理由**:
- 3ヶ月の開発期間を考慮し、10人～20人くらいを確保する。
- 実験時間を20-30分→15分に短縮
- 小規模でも統計的有意性は確保可能（各言語3名以上）
- 募集・調整の負担を軽減

#### 募集方法

1. **大学院の国際交流センター**に協力依頼
2. **留学生サークル**への告知
3. **SNS**（大学内グループ）での募集

#### 除外基準

- 日本語が流暢な留学生 → システムの必要性が低い
- IT関連専攻で技術的知識が豊富 → バイアスの可能性

### 2.2 実験環境

| 項目 | 仕様 |
|------|------|
| **実施場所** | 大学内の会議室 / 静かなカフェ |
| **使用デバイス** | 被験者自身のスマートフォン（Android推奨） |
| **アプリ** | Expo Goで実行 |
| **ネットワーク** | Wi-Fi環境（大学WiFi または モバイルホットスポット） |
| **所要時間** | 1人あたり15分（簡略化版） |

## 3. 評価実験の手順

### 3.1 実験フロー

```
1. 事前説明（2分）
   - 実験の目的説明
   - 簡易同意確認（口頭）
   ↓
2. デモグラフィック情報収集（1分）
   - 年齢、母語、日本語レベル
   ↓
3. タスク実施（5分）
   - QRコードスキャン
   - メニュー閲覧（母語で）
   - 料理選択・注文
   - タスク完了時間計測
   ↓
4. アンケート回答（5分）
   - 定量評価（5段階評価）
   - 定性評価（自由記述、簡潔版）
   ↓
5. 簡易インタビュー（2分）
   - 使用感のヒアリング（簡易版）
   - 主要な改善要望のみ収集
   ↓
6. 謝礼渡し・終了
```

**簡略化の理由**: 3ヶ月の開発期間を考慮し、実験時間を20-30分→15分に短縮。研究に必要な定量データは維持しつつ、詳細なインタビューを簡略化。

### 3.2 実施タスク詳細

#### タスク1: QRコードスキャン

**指示**: 「テーブル上のQRコードをスキャンしてください」

**測定項目**:
- タスク完了可否
- 所要時間（秒）
- エラー発生の有無

#### タスク2: 言語選択

**指示**: 「あなたの母語を選択してください」

**測定項目**:
- 言語選択の直感性
- UIの分かりやすさ

#### タスク3: メニュー閲覧

**指示**: 「メニューから好きな料理を3つ選んでカートに追加してください」

**測定項目**:
- メニュー名の理解度（5段階評価）
- 説明文の分かりやすさ（5段階評価）
- アレルゲン情報の視認性

#### タスク4: 注文実行

**指示**: 「カート内の商品を確認して注文を確定してください」

**測定項目**:
- タスク完了可否
- 所要時間（全タスク合計）

## 4. 定量評価

### 4.1 翻訳精度比較実験

#### 実験設計

| 比較対象 | 説明 |
|---------|------|
| **提案手法** | 専門用語辞書 + 翻訳API のハイブリッド |
| **既存手法** | 翻訳APIのみ（辞書なし） |

#### 評価セット

専門用語を含むメニュー項目20個を選定:

| No. | 原文（日本語） | カテゴリ |
|-----|--------------|---------|
| 1 | 炙りサーモン寿司 | 刺身・寿司 |
| 2 | 大トロ | 刺身・寿司 |
| 3 | 海鮮丼 | ご飯・麺 |
| 4 | 天ぷら盛り合わせ | 揚げ物 |
| 5 | 親子丼 | ご飯・麺 |
| ... | ... | ... |

#### 評価方法

1. **自動評価**: BLEU スコア（参照訳との一致度）
2. **人手評価**: ネイティブスピーカー（留学生）による評価

**評価基準**:
- ◎（5点）: 完璧な翻訳
- ○（4点）: ほぼ正確、ニュアンスも適切
- △（3点）: 意味は通じるが不自然
- ×（2点）: 誤訳あり、意味が不明瞭
- ××（1点）: 完全な誤訳

#### データ記録

[tables_template.md](./tables_template.md) の「翻訳精度比較テーブル」を使用。

### 4.2 タスク完了時間

| 測定項目 | 目標値 | 測定方法 |
|---------|-------|---------|
| **QRスキャン〜注文確定** | 3分以内 | ストップウォッチ |
| **メニュー選択時間** | 1分以内 | アプリ内ログ |

### 4.3 満足度スコア（5段階評価）

アンケート項目:

| No. | 質問項目 | 評価尺度 |
|-----|---------|---------|
| Q1 | アプリの使いやすさ | 1（とても使いにくい）〜 5（とても使いやすい） |
| Q2 | メニュー翻訳の分かりやすさ | 1（全く分からない）〜 5（完璧に理解できる） |
| Q3 | 料理名の正確さ | 1（不正確）〜 5（非常に正確） |
| Q4 | 説明文の自然さ | 1（不自然）〜 5（とても自然） |
| Q5 | アレルゲン情報の分かりやすさ | 1（分かりにくい）〜 5（とても分かりやすい） |
| Q6 | 全体的な満足度 | 1（不満）〜 5（大変満足） |
| Q7 | 実際の飲食店で使いたいか | 1（使いたくない）〜 5（ぜひ使いたい） |

### 4.4 成功基準

| 項目 | 目標値 |
|------|-------|
| **タスク完了率** | 90%以上 |
| **平均満足度** | 4.0以上（5段階） |
| **翻訳精度向上率** | 既存手法比20%向上 |
| **「使いたい」評価** | 70%以上が4点以上 |

## 5. 定性評価

### 5.1 自由記述アンケート

| 質問 | 意図 |
|------|------|
| 「このアプリの良かった点を教えてください」 | 強みの把握 |
| 「このアプリの改善すべき点を教えてください」 | 弱みの発見 |
| 「どんな機能があれば便利だと思いますか?」 | 機能拡張のヒント |
| 「実際の飲食店で使う場合、心配な点はありますか?」 | 実用化の課題抽出 |

### 5.2 フォローアップインタビュー

#### 質問例

1. **翻訳について**
   - 「メニューの翻訳で理解しにくかった料理はありましたか?」
   - 「料理の説明は十分でしたか?」

2. **UI/UXについて**
   - 「どの画面が使いにくかったですか?」
   - 「迷った操作はありましたか?」

3. **実用性について**
   - 「日本の飲食店で注文する際、困ることはありますか?」
   - 「このアプリがあれば、その問題は解決しますか?」

### 5.3 データ整理

[tables_template.md](./tables_template.md) の「アンケート自由記述整理テーブル」を使用。

## 6. データ収集・分析

### 6.1 収集データ一覧

| データ種別 | 収集方法 | 形式 |
|-----------|---------|------|
| **デモグラフィック** | 事前アンケート | Googleフォーム |
| **タスク完了時間** | ストップウォッチ計測 | Excelシート |
| **翻訳精度評価** | 人手評価 | CSVファイル |
| **満足度スコア** | 5段階評価アンケート | Googleフォーム |
| **自由記述** | アンケート | Googleフォーム |
| **インタビュー記録** | 音声録音（許可を得て） | テキスト起こし |
| **アプリ操作ログ** | Firebase Analytics | JSON |

### 6.2 分析手法

#### 定量データ

| 分析項目 | 手法 |
|---------|------|
| **翻訳精度比較** | 対応のあるt検定（提案手法 vs 既存手法） |
| **満足度分析** | 平均値・標準偏差の算出 |
| **タスク完了時間** | 記述統計（平均、中央値、範囲） |
| **言語別比較** | 英語圏 vs 中国語圏の差異分析 |

#### 定性データ

| 分析項目 | 手法 |
|---------|------|
| **自由記述** | テキストマイニング、カテゴリ分類 |
| **インタビュー** | KJ法でグルーピング |
| **問題点抽出** | 頻出する課題のリストアップ |

## 7. 倫理的配慮

### 7.1 インフォームドコンセント

実験開始前に以下を説明し、同意書に署名:

- 実験の目的
- データの使用用途（卒業研究のみ）
- 個人情報の保護（匿名化）
- 参加の任意性（いつでも中止可能）
- 実験中の録音・録画の有無

### 7.2 個人情報保護

- 被験者IDを割り当て、氏名は記録しない
- データは研究目的のみに使用
- 論文には統計データのみ記載、個人特定不可

### 7.3 同意書テンプレート

```
実験参加同意書

私は、以下について説明を受け、理解しました。

1. 本実験の目的は、多言語注文システムの評価である
2. 収集されたデータは卒業研究にのみ使用される
3. 個人情報は匿名化され、論文には掲載されない
4. 参加は任意であり、いつでも中止できる
5. 実験中、不快に感じた場合は申し出ることができる

上記を理解し、実験への参加に同意します。

日付:
被験者ID:
署名:
```

## 9. 想定される課題と対策

### 9.1 リスクと対策

| リスク | 対策 |
|-------|------|
| **被験者が集まらない** | 謝礼額の増額、募集期間の延長 |
| **システムのバグ** | 事前に十分なテスト実施 |
| **ネットワーク不安定** | モバイルホットスポットを準備 |
| **実験時間超過** | タスクの簡素化 |
| **翻訳精度が低い** | 辞書データの事前チューニング |

### 9.2 バックアッププラン

- **被験者不足**: 最低5名（英語2名、中国語3名）で実施
- **技術的トラブル**: 予備のスマートフォンを用意
- **時間不足**: アンケートをオンラインで後日回答

## 10. 期待される成果

### 10.1 定量的成果

- 翻訳精度: 既存手法比20%以上向上
- タスク完了率: 90%以上
- 満足度: 平均4.0以上（5段階）

### 10.2 定性的成果

- 実用化に向けた具体的な改善点の洗い出し
- 外国人観光客のニーズの明確化
- システムの強み・弱みの特定

### 10.3 論文への貢献

- 評価実験のデータを論文の「実験結果」セクションに記載
- 定量・定性データの両面から有効性を主張
- 実用化に向けた課題と展望を考察
