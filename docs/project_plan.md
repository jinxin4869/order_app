# QRコード対応多言語注文システム プロジェクト計画書

## 開発手法

- **フロントエンド**: React Native によるクロスプラットフォームアプリ開発
  - iOS / Android 対応
  - QRコードベースのテーブル認識システムにより、顧客が自身のスマートフォンから直接アクセス・注文
- **バックエンド / データベース**: Firebase
  - 認証、リアルタイムデータベース / Firestore、Cloud Functions 等
- **翻訳API**: 汎用翻訳APIを利用
  - 候補: Google Translate API, DeepL API など

## 翻訳精度向上手法

汎用翻訳APIだけでは飲食店特有の専門用語で誤訳が生じる可能性があるため、以下の仕組みで補完する。

1. **専門用語辞書の構築**
   - 対象: 料理名、食材、調理法、アレルギー関連用語
   - 多言語（日本語・英語・中国語など）対応の辞書データベースを作成
2. **前処理（入力テキスト処理）**
   - 自然言語処理ライブラリ（例: MeCab）による形態素解析
   - 解析結果と専門用語辞書をマッチングし、専門用語を識別
3. **翻訳API呼び出し時のコンテキスト付与**
   - 辞書で特定した専門用語を、翻訳APIに渡すテキストのコンテキストとして付与
   - 必要に応じて用語をプレースホルダ化して誤訳を抑制
4. **後処理（翻訳結果の補正）**
   - 翻訳結果中の対象用語を辞書に基づき再置換・補正
   - 専門用語が意図した表現になっているかを辞書ベースでチェック

## 実証実験手法

実際の小規模飲食店に試験導入し、外国人観光客および飲食店スタッフを対象とした実証実験を行う。

- **対象店舗**
  - 近隣の外国人利用頻度の高い飲食店（3店舗程度）
  - システムの実証実験で利用し価値を見出す可能性が高い店舗
- **対象者**
  - 外国人観光客
  - 飲食店スタッフ
- **目的**
  - 外国人観光客のメニュー理解度・注文容易さ・満足度の変化を評価
  - 飲食店の注文受付業務効率性・スタッフ負担感の変化を評価

## 評価手法

### 定量評価

- 注文処理時間の測定（注文開始〜注文確定までの時間）
- システム利用率の分析
  - 来店客数に対するシステム利用客の割合
  - 時間帯別利用状況
- 顧客満足度の比較分析
  - アンケート（5段階評価など）による前後比較

### 定性評価

- 飲食店スタッフへのインタビュー調査
  - 業務効率の変化
  - 注文ミスの減少やコミュニケーション負担の変化
  - システムの使いやすさ・改善要望
- 外国人観光客へのインタビュー・自由記述アンケート
  - メニュー理解のしやすさ
  - 自国語での注文体験の満足度

## システム構成

### フロントエンド

- React Native アプリ
  - QRコード読み取り機能（テーブル認識）
  - 多言語対応チャットボットUI
    - メニュー閲覧
    - 数量選択
    - 注文カート機能

### バックエンド

- Firebase を中心としたサーバーレス構成
  - 注文処理ロジック（Cloud Functions 等）
  - 在庫管理
  - 注文ステータス管理

### 管理インターフェース

- 飲食店向け Web ダッシュボード
  - 新規注文の通知表示
  - 注文内容の確認
  - 売上集計・期間別レポート

### 専門用語辞書

- 料理名・食材・調理法・アレルギー関連用語などを含むマルチリンガル辞書データベース
- フロントエンド・バックエンドから参照し、翻訳APIとの連携や表示テキスト補正に利用

# プロジェクト進行の概要ステップ

1. 要件定義・詳細設計
2. 専門用語辞書の設計と初期データ作成
3. React Native フロントエンドのプロトタイプ開発
4. Firebase バックエンドと管理ダッシュボードの構築
5. 翻訳API + 辞書連携ロジックの実装
6. 店舗での実証実験準備
7. 実証実験の実施とデータ収集
8. 評価・改善サイクル
